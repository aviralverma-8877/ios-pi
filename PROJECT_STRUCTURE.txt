🎯 PURPOSE OF EACH DIRECTORY:

   📁 ROOT (/)
      - Contains core application and project-level files
      - Main entry point: pi_ios_desktop.py
      - Project documentation: README.md
      - License and configuration files

   📁 deb-build/
      - Everything needed to build Debian packages
      - Build script with packaging logic
      - Session files required by package
      - Self-contained build environment

   📁 scripts/
      - User-facing installation and management scripts
      - Manual installation (without .deb package)
      - Uninstallation and cleanup
      - System configuration helpers

   📁 docs/
      - All documentation files
      - Getting started guides
      - Detailed installation instructions
      - Troubleshooting and reference materials

═══════════════════════════════════════════════════════════════

🚀 USAGE EXAMPLES:

   Building Debian Package:
   $ cd deb-build
   $ ./build-desktop-deb.sh
   $ cd ..
   $ sudo dpkg -i pi-ios-desktop_1.0.0_all.deb

   Manual Installation:
   $ cd scripts
   $ sudo ./setup-desktop-environment.sh

   Running Directly (Testing):
   $ python3 pi_ios_desktop.py

   Reading Documentation:
   $ cat docs/START-HERE.md
   $ less docs/INSTALL.md

   Uninstalling:
   $ cd scripts
   $ sudo ./uninstall-desktop.sh

✅ BENEFITS OF THIS STRUCTURE:

   ✓ Clear separation of concerns
   ✓ Easy to navigate and understand
   ✓ Build files isolated from runtime files
   ✓ Documentation organized in one place
   ✓ Scripts can be run from their directories
   ✓ Clean root directory
   ✓ Professional project structure
   ✓ Easy to package and distribute
   ✓ Git-friendly with proper .gitignore

═══════════════════════════════════════════════════════════════

📝 NOTES FOR DEVELOPERS:

   • All scripts use dynamic path resolution (SCRIPT_DIR, PROJECT_ROOT)
   • Scripts can be run from any directory
   • Paths are relative to script location
   • No hard-coded absolute paths
   • Build output (.deb) goes to project root
   • Temporary build files in deb-build/debian-pkg/
